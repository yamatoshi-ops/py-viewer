# Proto #0.3 設計書

---

## 1. 目的

同一行への複数チャンネル重ね表示を実現する。
テキストエリアで行グルーピングを定義する方式（案E）の検証プロトタイプ。

---

## 2. 機能一覧

| # | 機能 | #0.2 との差分 |
|---|------|--------------|
| 1 | **1行複数チャンネル重ね表示** | **NEW** — カンマ区切りで同一行に複数チャンネルを重ねて描画 |
| 2 | **テキストエリアによるレイアウト定義** | **NEW** — 1行=1波形行、カンマ区切りでグルーピング |
| 3 | **チャンネル色分け** | **NEW** — 8色パレットで自動割り当て |
| 4 | **利用可能チャンネル一覧表示** | **NEW** — データから自動検出したチャンネル名を表示 |
| 5 | 時間軸同期（shared_xaxes） | #0.1 から継続 |
| 6 | ホバー（x unified） | #0.1 ベース（#0.2 の行レイアウト方式ではない） |
| 7 | スクロールズーム | #0.1 から継続 |

## 3. 使用方法

### 起動

```bash
cd proto_0_3
source .venv/bin/activate
python app.py
# → http://localhost:8050
```

### サンプルデータ生成

```bash
python generate_sample.py
```

モータ制御ステップ応答をシミュレートしたサンプルデータを生成する。

| チャンネル | 内容 |
|-----------|------|
| id_ref | d軸電流指令（ステップ入力 5A @ t=0.02s） |
| id | d軸電流（一次遅れ応答 τ=5ms + ノイズ） |
| iq_ref | q軸電流指令（ステップ入力 10A @ t=0.01s） |
| iq | q軸電流（一次遅れ応答 τ=5ms + ノイズ） |
| voltage_u | U相電圧（200V / 50Hz 正弦波） |
| voltage_v | V相電圧（200V / 50Hz 正弦波、-120°位相） |

パラメータ: 10kHz サンプリング、100ms、1,000点。

### 操作方法

| 操作 | 動作 |
|------|------|
| テキストエリアを編集 | 行グルーピングを変更（1行=1波形行、カンマ区切りで重ね表示） |
| 「更新」ボタン | 編集したレイアウトを波形に反映 |
| ドラッグ | 時間軸をパン |
| マウスホイール | 時間軸をズーム |
| ダブルクリック | ズームリセット |

### レイアウト記法

```
id_ref, id        ← id_ref と id を同じ行に重ねて表示
iq_ref, iq        ← iq_ref と iq を同じ行に重ねて表示
voltage_u         ← voltage_u を単独行で表示
voltage_v         ← voltage_v を単独行で表示
```

- 1行 = 1波形行
- カンマ区切りでチャンネルをグルーピング
- 3チャンネル以上の重ねも可能（例: `id_ref, id, iq`）
- データに存在しないチャンネル名は無視される

```
│ レイアウト定義             │ 利用可能チャンネル      │

│ └───────────────────────┘ │                        │

├───────────────────────────┴────────────────────────┤
│  行1: id_ref (青) / id (オレンジ) ── 重ね表示       │
│  ─────────────────────────────────────────────────  │
│  行2: iq_ref (緑) / iq (赤) ── 重ね表示            │
│  ─────────────────────────────────────────────────  │
│  行3: voltage_u ── 単独表示                         │
│  ─────────────────────────────────────────────────  │
│  行4: voltage_v ── 単独表示                         │
└────────────────────────────────────────────────────┘
```

### 主要関数

**`parse_layout(text: str) -> list[list[str]]`**

テキストから行グルーピングをパースする。

- 空行をスキップ
- カンマで分割し、前後の空白を除去
- データに存在するチャンネルのみ残す

```python
parse_layout("id_ref, id\niq_ref, iq\nvoltage_u")
# → [["id_ref", "id"], ["iq_ref", "iq"], ["voltage_u"]]
```

**`build_figure(row_groups: list[list[str]]) -> go.Figure`**

行グルーピングに従って `make_subplots` でFigureを生成する。

- `shared_xaxes=True` で時間軸同期
- 同一行の複数チャンネルは同じ subplot に `add_trace`
- Y軸ラベルは `" / ".join(group)`（例: "id_ref / id"）
- 色は `TRACE_COLORS[j % 8]` で自動割り当て

### Callback フロー

```
[「更新」ボタン クリック]
      │
      ▼
  update_graph(n_clicks, layout_text)
    Input:  update-btn.n_clicks
    State:  layout-input.value
      │
      ├── parse_layout(layout_text) → row_groups
      ├── build_figure(row_groups)  → fig
      └── Output: graph-container.children ← dcc.Graph(figure=fig)
```

シンプルな1コールバック構成。#0.2 のホバー同期・ズーム同期は含まない（#0.1 ベース）。

### 色パレット

```python
TRACE_COLORS = [
    "#1f77b4",  # 青
    "#ff7f0e",  # オレンジ
    "#2ca02c",  # 緑
    "#d62728",  # 赤
    "#9467bd",  # 紫
    "#8c564b",  # 茶
    "#e377c2",  # ピンク
    "#7f7f7f",  # グレー
]
```

同一行内のチャンネルにインデックス順で割り当て。行をまたいでリセットされる。

## 6. 検証結果と次のステップ

テキストエリア方式は動作するが、以下の課題がある:

- チャンネル名を手入力する必要があり、タイプミスのリスクがある
- チャンネル数が多い場合、テキスト管理が煩雑になる

→ **Proto #0.4** でドロップダウンUI方式を検証する（完了済み）。
→ UI方式が固まったら **Proto #1** へ移植する。
