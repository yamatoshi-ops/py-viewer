# Panel_y 波形表示機能 開発計画書

## 1. 概要

Proto #1（実用ビューア）をベースに、波形表示・解析機能を段階的に強化する。
プロトタイプでの検証を経てから #1 へ移植する方針を維持する。

## 2. 開発フェーズ

### Phase 1: 表示機能の強化

基本的な見た目・操作性を #1 に揃える。

| # | 機能 | 内容 | 実装先 | 状態 |
|---|------|------|--------|------|
| 1-1 | 1行複数チャンネル表示 | #0.4 のドロップダウンUI方式を #1 へ移植 | #2.0 | 完了 |
| 1-2 | line/step 表示切り替え | チャンネルごとに plot（線）/ stairs（階段）を選択。デジタル信号・PWMゲート信号向け | #1 | 未着手 |
| 1-3 | 線の色・太さ選択 | デフォルトはパレット自動割り当て。変更したいチャンネルだけ個別指定 | #1 | 未着手 |

**技術メモ:**
- step表示: `go.Scatter(line_shape="hv")` で実現
- 色: `dcc.Input(type="color")` またはプリセットパレット
- 太さ: 数値入力 or スライダー（デフォルト1px）
- チャンネルごとの表示設定は、行UIのドロップダウン横にコンパクトに配置する

---

### Phase 2: 大容量データ対応

実機ログ（数十万〜数百万点）を快適に表示するための間引き機構。

| # | 機能 | 内容 | 実装先 | 状態 |
|---|------|------|--------|------|
| 2-1 | LOD（Level of Detail）間引き | 表示範囲のデータ点数が閾値を超えたら間引いて描画。ズームインで解像度が上がり、最大拡大で生データ | #1 | 未着手 |

**技術メモ:**
- `relayoutData` から `xaxis.range` を取得 → 表示範囲のデータ点数を計算
- 閾値: 画面幅のピクセル数程度（目安 3,000〜5,000点）
- 間引きアルゴリズム: **LTTB（Largest Triangle Three Buckets）**
  - 等間隔間引きだとピーク（過渡応答・スパイク）が消えるリスクがある
  - LTTBは視覚的に重要な点を残す。NumPyで数十行で実装可能
- **重要**: 解析系（カーソル値読み取り・FFT）は常に生データを参照する設計にする

---

### Phase 3: カーソル解析

Proto #1 の Cursor A/B を拡張し、複数カーソルによる区間解析を実現。

| # | 機能 | 内容 | 実装先 | 状態 |
|---|------|------|--------|------|
| 3-1 | 複数カーソル配置 | T1, T2, T3... を任意数配置。クリックで追加、右クリックまたはボタンで削除 | #1 | 未着手 |
| 3-2 | カーソル値表示 | 各カーソル位置での全チャンネル値をパネル表示 | #1 | 未着手 |
| 3-3 | 区間統計 | 任意の2カーソル間（例: T1-T2）の統計値を表示 | #1 | 未着手 |

**3-3 区間統計の表示項目:**
- 時間間隔 Δt
- 平均値 (Mean)
- 最大値 (Max) / 最小値 (Min)
- 傾き (Δy/Δt) — モータの加速度推定等に有用
- RMS（実効値）

**技術メモ:**
- カーソル位置は `dcc.Store` でリスト管理
- 区間統計は DataFrame スライス → NumPy 演算（計算自体は軽量）
- 2カーソル選択UIは、ドロップダウン（T1, T2, ...から2つ選択）が明確

---

### Phase 4: 信号処理

波形に対してリアルタイムにフィルタ・変換を適用し、加工チャンネルとして追加表示する。

| # | 機能 | 内容 | 実装先 | 状態 |
|---|------|------|--------|------|
| 4-1 | ゼロ位相LPF（filtfilt） | 指定カットオフ周波数でLPFを適用。元信号を残したまま `{ch}_lpf` として追加 | #1 | 未着手 |
| 4-2 | FFT / DFT | 2カーソル間の信号をFFT解析。スペクトルを別パネルで表示 | #1 | 未着手 |

**技術メモ:**
- LPF: `scipy.signal.filtfilt` + Butterworth フィルタ（`scipy.signal.butter`）
  - パラメータ: カットオフ周波数、次数（デフォルト2次で十分）
  - MATLABの `filtfilt` と同等のゼロ位相特性
- FFT: `np.fft.fft` → 片側スペクトル計算
  - 窓関数の選択（Hanning, Hamming, Rectangular）があると実用的
  - 横軸: 周波数 [Hz]、縦軸: 振幅 or dB
- 加工チャンネルは元データに追加する形。チャンネル選択ドロップダウンに自動追加される

**将来候補（未計画）:**
- THD（全高調波歪み率）・高調波次数解析
- HPF / BPF / ノッチフィルタ
- 移動平均・エンベロープ検出

### Phase 5: コマンド呼び出し機能

GUIでは煩雑になる高度な操作を、ターミナル風のコマンド入力で実行する。

| # | 機能 | 内容 | 実装先 | 状態 |
|---|------|------|--------|------|
| 5-1 | コマンドパーサー | テキスト入力 + Enter でコマンドを解釈・実行 | #1 | 未着手 |
| 5-2 | 解析コマンド群 | Phase 3-4 の機能をコマンドから呼び出し | #1 | 未着手 |

**コマンド例（案）:**
```
fft T1 T2 ch=iq              # T1-T2区間のiq信号をFFT
lpf id fc=1000                # id にカットオフ1kHzのLPFを適用
stats T1 T2                   # T1-T2区間の統計値を表示
cursor add 0.025              # t=0.025s にカーソルを追加
```

**技術メモ:**
- `dcc.Input` + Enter キーで発火
- コマンド文字列をパースして対応する処理関数を呼び出す
- コマンド履歴（↑↓キー）があると使いやすい
- **棲み分け**: 基本操作（カーソル配置、チャンネル選択）はGUI、高度な解析はコマンド

## 3. 着手順と依存関係

```
Phase 1: 表示強化  ←  #0.4 検証済み、すぐ着手可能
    │
Phase 2: 間引き    ←  独立して着手可能
    │
Phase 3: カーソル  ←  Phase 2 の後が望ましい（間引き時も生データ参照の仕組みが必要）
    │
Phase 4: 信号処理  ←  Phase 3 の区間選択UIを利用
    │
Phase 5: コマンド  ←  Phase 3-4 の上に載せるフロントエンド
```

Phase 1 と Phase 2 は独立しているため並行可能。
Phase 3 以降は順序依存があるため逐次進行。

## 4. プロトタイプ検証方針

| 機能 | プロトタイプで検証するか | 理由 |
|------|------------------------|------|
| 1行複数チャンネル | 済（#0.3, #0.4） | UI方式の比較が必要だった |
| line/step 切り替え | 不要 | Plotlyの `line_shape` 変更のみ。#1 で直接実装 |
| 色・太さ選択 | 不要 | UI部品の組み合わせのみ |
| LOD 間引き | 要検討 | LTTBの挙動確認は #0.5 等で小さく試す価値あり |
| カーソル解析 | 不要 | #1 に既存のA/B方式があり、拡張で対応可能 |
| 信号処理 | 要検討 | filtfilt の動作確認は #1 上でインクリメンタルに進められる |
| コマンド機能 | 不要 | パーサー自体はシンプル。#1 上で直接実装 |
